# 🧬 회고 및 교리 진화 프로토콜 (Retrospective & Doctrine Evolution)

현재 작업 세션은 종료되었습니다. 이제 당신의 역할은 **'메타 아키텍트(Meta-Architect)'**로 전환됩니다.
당신의 임무는 방금 수행한 작업을 비판적으로 분석하고, 미래의 에이전트(미래의 당신)가 더 잘 수행할 수 있도록 **영구적인 규칙(Rules)**을 생성하거나 업데이트하는 것입니다.

## 단계 1: 세션 분석 (Internal Analysis)
방금 수행한 대화와 작업 내역을 처음부터 끝까지 검토하십시오.
- **성공 요인:** 무엇이 잘 작동했습니까?
- **실패 및 수정:** 사용자가 당신의 코드를 수정했거나, 에러가 발생한 지점은 어디입니까? **근본 원인(Root Cause)**은 무엇입니까?

## 단계 2: 교훈 추출 (Lesson Distillation)
분석을 바탕으로 **'보편적인 원칙(Universal Principle)'**을 추출하십시오.
- (나쁜 예): "app.py의 50번째 줄을 수정했다." (너무 구체적임)
- (좋은 예): "DB 연결 함수는 반드시 타임아웃 처리를 포함해야 한다." (보편적임, 규칙으로 만들 수 있음)

## 단계 3: 규칙 업데이트 (Integration)
추출한 교훈을 바탕으로 다음 파일 중 하나를 업데이트하는 **구체적인 파일 수정(Diff)**을 제안하십시오.
1. `.agent/rules/DOCTRINE.md`: 전역적인 행동 강령이 필요한 경우.
2. `.agent/rules/<tech-stack>.md`: 특정 기술(예: React, Python)에 대한 구체적 테크닉인 경우.
3. `.agent/memory/CODEBASE_MAP.md`: 프로젝트 구조나 주요 파일 위치가 변경된 경우.

## 🛑 출력 형식 (반드시 준수)
1. **분석 요약:** (3줄 이내)
2. **발견된 문제점:** (무엇을 놓쳤는가?)
3. **제안된 규칙 업데이트:** (어느 파일에 어떤 문구를 추가/수정할 것인가?)
