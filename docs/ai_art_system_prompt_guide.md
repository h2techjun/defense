## 🎯 1. 시스템 프롬프트

> **모든 배경 생성 요청 시 반드시 맨 앞에 포함**하는 프롬프트입니다.  
> 이 프롬프트가 전체 배경의 스타일을 고정하는 앵커(anchor) 역할을 합니다.

### 전체 시스템 프롬프트

아래 전문을 **그대로** 복사하여 이미지 생성 모델에 시스템 프롬프트(또는 프롬프트 맨 앞)로 넣으세요.

```
[시스템 프롬프트 — 배경 아트 디렉터]

당신은 "해원의 문: Soul Defense"의 배경 아트 디렉터입니다.
한국 전통 판타지 모바일 타워 디펜스 게임의 배경 이미지를 제작합니다.

■ 시각적 정체성
- 화풍: 한국 전통 수묵화(水墨畫) 텍스처 + 현대 게임 아트 마감
- 분위기: 다크 판타지 — 어둡지만 아름답고, 신비롭지만 쓸쓸한 느낌
- 구도: 16:9 가로 풍경 구도 (UI 요소가 위에 올라갈 공간 고려)
- 광원: 부드러운 글로우(glow) 악센트 — 한두 군데 빛나는 포인트
- 질감: 붓터치와 먹번짐 느낌이 은은하게 남아있는 페인터리(painterly) 스타일

■ 색상 규칙
- 기본 톤: 짙은 남색(#0D001A) ~ 먹색(#1A1A2E) 베이스
- 따뜻한 악센트: 금빛(#FFD700), 코랄(#FF6B6B), 호박색(#FF9F43)
- 차가운 악센트: 도깨비불 청색(#4ECDC4), 영혼 보라(#A855F7)
- 밝은 영역: 달빛 백색(#F0E6FF), 벚꽃 핑크(#FFB7C5)
- 그림자: 순수 검정(#000000) 대신 짙은 보라-갈색 사용

■ 한국 전통 요소 (반드시 하나 이상 포함)
- 기와 (giwa tiles) — 곡선형 지붕 패턴
- 단청 (dancheong) — 처마 밑 오방색 문양
- 부적 (talismans) — 금빛으로 빛나는 한자 부적
- 장승 (jangseung) — 수호 토템
- 솟대 (sotdae) — 새가 올라간 기둥
- 홍살문 (hongsal-mun) — 붉은 문
- 한옥 (hanok) — 전통 가옥
- 피안화 (spider lily) — 저승꽃
- 도깨비불 (ghost fire) — 파란 유령 불꽃

■ 절대 규칙
- ❌ 사실적(realistic) / 사진(photographic) 스타일 금지
- ❌ 서양 판타지 요소 (성, 기사, 드래곤) 금지
- ❌ 일본풍(anime/manga) 직접 참조 금지
- ❌ 캐릭터/인물 직접 그리기 금지 (실루엣은 허용)
- ❌ 글자/워터마크/프레임 금지
- ✅ 한국 민화(folk painting) + 게임 아트 퓨전
- ✅ UI 오버레이 공간 확보 (상단/하단 여백)
- ✅ 깨끗한 고해상도 출력
```

### 간결 버전 (토큰 제한 시)

```
한국 전통 판타지 게임 배경. 수묵화 텍스처 + 다크 판타지 분위기.
부드러운 글로우 악센트, 페인터리 스타일. 16:9 가로. 게임 아트 품질.
남색-먹색 베이스 + 금빛/도깨비불 악센트. 한국 전통 건축/자연 요소 포함.
캐릭터 없음, 글자 없음. UI 오버레이 공간 확보.
```

## 🗺️ 3. 맵 배경 전략

### 3-1. 핵심 질문: 맵 배경을 어떻게 만들 것인가?

맵 배경은 UI 배경과 다르게 **게임 플레이 중 계속 보이는** 배경입니다.
타워 디펜스에서 맵 배경은 길(path), 타워 배치 슬롯, 적의 이동 경로를 모두 포함해야 합니다.

> [!IMPORTANT]
> 맵 배경은 AI 이미지 하나로 완성하기 어렵습니다.
> 가장 현실적인 접근은 **기본 분위기 배경 + 코드 오버레이** 조합입니다.

### 3-2. 3가지 접근 방식

#### 방안 A: 1배경 + 코드 컬러 필터 (최소 작업량 ⚡)

**원리**: 에피소드당 기본 배경 1장만 생성하고, 코드로 색상 오버레이를 적용하여 변형

```
에피소드 1 기본 배경 1장
  ├── ☀️ 낮:  원본 그대로
  ├── 🌅 석양: Color(0x26FF6400) 오버레이
  ├── 🌙 밤:  Color(0x66000050) 오버레이 + 도깨비불 파티클
  └── 🔴 보스: Color(0x4DB40000) 오버레이 + 화면 흔들림
```

| 장점                    | 단점                 |
| ----------------------- | -------------------- |
| 에피소드당 1장 (총 5장) | 분위기 차이가 약함   |
| 구현이 간단             | 밤/낮 사물 변화 없음 |
| 용량 최소               |                      |

**Dart 구현 예시**:

```dart
// 색상 오버레이 적용
class MapBackground extends SpriteComponent {
  void applyTimeFilter(MapTimeOfDay time) {
    switch (time) {
      case MapTimeOfDay.day:    paint.colorFilter = null;
      case MapTimeOfDay.sunset: paint.colorFilter = ColorFilter.mode(
        Color(0x26FF6400), BlendMode.overlay);
      case MapTimeOfDay.night:  paint.colorFilter = ColorFilter.mode(
        Color(0x66000050), BlendMode.multiply);
      case MapTimeOfDay.boss:   paint.colorFilter = ColorFilter.mode(
        Color(0x4DB40000), BlendMode.overlay);
    }
  }
}
```

---

#### 방안 B: 에피소드당 4변형 이미지 (권장 ⭐)

**원리**: 에피소드별로 낮/석양/밤/보스 4장을 별도로 생성

```
총 이미지: 에피소드 5 × 변형 4 = 20장
```

| 장점                       | 단점             |
| -------------------------- | ---------------- |
| 풍부한 분위기 연출         | 20장 생성 필요   |
| 각 변형마다 세부 변화 가능 | 시간과 비용 소요 |
| 가장 밸런스 좋은 선택      |                  |

> 각 변형의 프롬프트는 아래 [3-4. 에피소드별 맵 프롬프트]에 정리되어 있습니다.

---

#### 방안 C: 멀티 레이어 패럴랙스 (최고 품질 🏗️)

**원리**: 배경을 3~4개 레이어로 분리하여 깊이감 있는 패럴랙스 스크롤 효과 구현

```
레이어 구조:
  Layer 0 (최후방): 하늘/산/달  — 느리게 스크롤
  Layer 1 (중간):   나무/건물    — 보통 스크롤
  Layer 2 (전면):   길/울타리    — 빠르게 스크롤
  Layer 3 (오버레이): 안개/파티클 — 코드로 처리
```

| 장점                  | 단점                      |
| --------------------- | ------------------------- |
| 가장 풍부한 시각 경험 | 레이어별 이미지 제작 필요 |
| 게임 느낌 극대화      | 구현 복잡도 높음          |
|                       | Phase 3 이후 적용 추천    |

## 📐 4. 기술 사양

### 이미지 사양 표

| 항목      | UI 배경             | 맵 배경                 |
| --------- | ------------------- | ----------------------- |
| 해상도    | 1920×1080           | 1280×720 또는 1920×1080 |
| 비율      | 16:9                | 16:9                    |
| 포맷      | PNG                 | PNG                     |
| 배경      | 불투명 (풀 배경)    | 불투명 (풀 배경)        |
| 색상 모드 | sRGB                | sRGB                    |
| 파일 크기 | 각 1MB 이하 권장    | 각 1MB 이하 권장        |
| 저장 경로 | `assets/images/bg/` | `assets/images/bg/`     |

### 파일명 규칙

```
UI 배경:   bg_{페이지명}.png        → bg_main_menu.png
맵 배경:   bg_map_ep{N}_{시간}.png  → bg_map_ep1_day.png
```

### UI 오버레이 안전 영역

```
┌─────────────────────────────────────────┐
│  상단 여백 (제목/상태바): 약 15%        │
```
